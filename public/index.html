Introduction

Welcome to my Three.js playground!

Let's cover a quick introduction of how this playground works, then we can look at the basics of Three.js:

Code / Project Structure

Each example from the previous menu lives in public/scripts/examples and is loaded as a module using RequireJS. The core setup and rendering code is initialized in public/scripts/main.js and passed to each example's constructor. I'd encourage you to open up these parts in your editor so you can take a peek at what's going on as we step through each part of Three.js.

NOTE: The goal of this project is to provide a Three.js playground with examples of each piece of functionality that makes learning and understanding the library as easy as possible. This hack-job of a sandbox is atypical of any and all best practices; however, I've structured it in a way that makes viewing Three.js examples in isolation easy and clear as possible. Reccomendations and contributions are welcome on GitHub.
